Kde je hostovaný váš web
a jak je zabezpečený?

Gabriela Janýšková
14. července 2026
5 min čtení

Jedna z prvních otázek, kterou dostávám od klientů: „Kde ten web vlastně bude?" Druhá hned za ní: „A co když ho někdo hackne?" Obě jsou dobré. Odpovím na ně najednou.

Cloudflare Pages — co to je?

Každý web, který stavím, hostuju na Cloudflare Pages. Cloudflare je americká firma, která provozuje jednu z největších síťových infrastruktur na světě. Denně přes ni prochází zhruba 20 % veškerého internetového provozu.

Cloudflare má přes 300 datových center rozmístěných po celém světě — jedno z nich je v Praze. Když si někdo z Česka otevře váš web, data nepřijdou ze serveru v Americe, ale přímo z Prahy. Proto se web načte tak rychle.

Musíte za hosting platit zvlášť?

Ne. Hosting na Cloudflare Pages je součástí ceny tvorby webu a neplatíte za něj nic navíc — ani měsíčně, ani ročně. Cloudflare Pages nabízí bezplatný plán pro statické weby, který pokryje provoz jakéhokoli malého a středního webu bez omezení.

Jedinou věc, za kterou platíte, je doména — tedy vaše adresa na internetu, třeba vasnazev.cz. To vychází přibližně na 200–250 Kč ročně.

Srovnání s klasickým hostingem: Sdílený hosting u českých poskytovatelů (Wedos, Forpsi, Active24) stojí typicky 500–2 000 Kč ročně. Cloudflare Pages je zdarma a zároveň rychlejší a bezpečnější.

HTTPS — zelený zámek a co za ním stojí

Každý web, který postavím, má automaticky HTTPS — zabezpečené připojení. Poznáte ho podle zámku v adresním řádku prohlížeče. To znamená, že veškerá komunikace mezi webem a návštěvníkem je šifrovaná.

Cloudflare certifikát vydává automaticky a bezplatně přes Let's Encrypt. Nemusíte nic nastavovat, prodlužovat ani platit — certifikát se obnovuje sám každé tři měsíce na pozadí.

Proč je HTTPS důležité? Tři důvody:

  • Google zvýhodňuje HTTPS weby ve výsledcích vyhledávání — web bez HTTPS dostane penalizaci
  • Prohlížeče zobrazují varování u stránek bez HTTPS — návštěvníci se leknou a odejdou
  • Data jsou šifrovaná — nikdo na cestě mezi serverem a prohlížečem nemůže data odposlouchávat

DDoS ochrana — útok, který váš web nepocítí

DDoS útok je situace, kdy na váš web najednou přijde obrovské množství falešných požadavků s cílem ho zahltit a shodit. U klasického hostingu to funguje — server prostě nestíhá a web přestane odpovídat.

Na Cloudflare Pages to nefunguje. Cloudflare provozuje vlastní DDoS ochranu, která je automaticky aktivní pro každý web na jejich síti. Útok je zachycen a filtrován ještě předtím, než se dostane k samotnému webu. Návštěvníci nic nezpozorují.

Reálný příklad: Cloudflare mitiguje průměrně přes 170 miliard hrozeb denně napříč všemi weby na jejich síti. To je kapacita, kterou žádný český hosting ani zdaleka nemá.

Proč čistý kód = nejbezpečnější web

Tohle je klíčový bod, který WordPress weby nikdy nemůžou nabídnout.

WordPress web je složený ze stovek souborů — samotný WordPress, téma, a typicky 10–30 pluginů. Každý z těchto pluginů je software, který někdo napsal, a software má zranitelnosti. Útočníci tyto zranitelnosti aktivně hledají a zneužívají. Proto se WordPress weby hackují — doslova každý den.

Jak to vypadá v praxi

Klient přijde s WordPressem, který byl hacknutý. Web přesměrovával na podvodné stránky, Google ho označil jako nebezpečný a výsledkem byl výpadek podnikání na několik dní. Příčina: zastaralý plugin pro formuláře, aktualizovaný naposledy před dvěma lety.

Weby, které stavím, jsou čistý statický kód — HTML, CSS a minimální JavaScript. Neexistuje žádná databáze, žádný redakční systém, žádné pluginy. Útočník nemá co napadnout. Doslova neexistuje vstupní bod, přes který by se mohl dostat dovnitř.

Co to znamená pro zákazníky vašeho webu

Zákazník, který přijde na váš web, dostane:

  • Zabezpečené HTTPS připojení (ten zelený zámek) — jeho prohlížeč mu řekne, že je vše v pořádku
  • Web načtený z nejbližšího serveru — bez čekání
  • Žádné varování od prohlížeče o nebezpečném webu
  • Jistotu, že web je online — nejezdí na přetíženém sdíleném serveru

Pozor na levný hosting: Sdílené hostingy znamenají, že na jednom serveru běží stovky webů najednou. Pokud jeden z nich dostane velkou návštěvnost nebo je napadený, může to zpomalit nebo shodit i váš web. S Cloudflare Pages tenhle problém neexistuje — každý web má vlastní izolované prostředí.

  • HTTPS certifikát — šifrované připojení, zelený zámek, automatické obnovy
  • DDoS ochrana — automaticky, bez příplatku, průmyslová kapacita
